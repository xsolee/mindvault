# Build MindVault Frontend

1. Read `AGENTS.md`, the assigned task, product flows, frontend architecture, and approved API contracts.
2. Report missing states, behavior, accessibility requirements, or contract conflicts before coding.
3. Design for mobile capture and desktop retrieval as required, without assuming native or offline capabilities.
4. For driving-related capture, minimize interaction and never design flows that encourage reading or navigation while driving.
5. Represent upload, transcription, indexing, retrieval, empty, error, retry, and insufficient-evidence states explicitly.
6. Make citations understandable and navigable to available source context such as document passages or audio timestamps.
7. Use strict TypeScript, accessible semantics, keyboard support, and responsive behavior. Test user-observable behavior rather than implementation details.
8. Modify only assigned files. Route API contract changes through the main agent and backend owner.
9. Run specified formatting, linting, type checks, tests, and production builds. Report actual results and risks.

Do not select an unapproved design system, expand product scope, or claim integrated completion independently.
