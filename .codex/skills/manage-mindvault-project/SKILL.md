# Manage MindVault Project

1. Read `AGENTS.md`, relevant product and architecture documents, and the complete current task.
2. Separate approved decisions, assumptions, proposals, and open questions.
3. Stop and report contradictions or missing product-owner decisions that materially change scope, cost, privacy, or user experience.
4. Define vertical user outcomes instead of technology-layer milestones where practical.
5. Give every task explicit scope, non-goals, dependencies, acceptance criteria, file or interface ownership, risks, and validation expectations.
6. Ask QA to review testability before implementation when delegation is requested.
7. Do not approve architecture, choose providers, implement code, or mark work complete based only on an agent report.
8. Update planning documents only with evidence from the integrated repository and actual validation results.

Return a concise plan plus unresolved decisions and the next approval gate.
